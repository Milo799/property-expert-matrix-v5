## 🏢 项目简介

本项目是一个**企业级多智能体 (Multi-Agent) 业务专家技能库**，专为物业管理行业数字化转型设计。团队汇聚了 **17 位** 拥有 10 年以上头部物业企业（万物云、龙湖、碧桂园等）实战经验的专家角色。

**核心演进 (V7.2)**:
*   🏛️ **法律合规红线 (V7.0.2/V7.0.3)**: 全面植入民法典 (公共收益分账/买卖不破租赁/优先购买权)、消防法 (第一责任人)、特种设备安全法 (15 天强制维保)、劳动法 (派遣<10% 红线)、环保法 (垃圾分类台账)、电子投票存证 (区块链上链)。
*   🏗️ **行业标准校准 (V7.0.1)**: 确立"四级房产结构 + 楼层独立字段"行业基线，剔除主数据治理/数据迁移等技术冗余。
*   🧹 **行业去伪存真 (V7.0.2)**: IOT 专家剔除 IT 协议术语 (RabbitMQ/MQTT)，回归物业业务场景；SUP 专家剔除制造业 VMI/EOQ，转为物业服务采购；PMA 增加物业接管验收/双轨制切换。
*   🔗 **质询网络完善 (V7.2)**: 补全 SVC→FIN/ASSET、ENG→ASSET、GOV→ASSET、COM→ASSET 等 5 条关键质询链路，消除业务盲区。
*   📂 **独立绩效归档**: 建立 `reports/` 目录，每日绩效日报独立生成，保持主文档整洁。
*   🏗️ **知识卡片化**: 17 位专家全部浓缩为高密度知识卡片 (表格/公式/符号)，Token 缩减 20%，信息密度提升 300%+。

## 🚀 核心优势

### 1. 🏗️ Map-Reduce 架构
摒弃低效的"全员大锅饭"会议，采用**动态按需加载**。仅调度 3-5 位相关专家**并发执行**，大幅降低 Token 消耗，消除上下文溢出风险。

### 2. 🏛️ 法律合规红线体系
*   **民法典**: 公共收益独立分账/买卖不破租赁/优先购买权/电子投票存证。
*   **消防法**: 项目经理第一责任人/月度防火检查/疏散演练台账/双人值班。
*   **特种设备法**: 电梯每 15 天强制维保/锅炉年检/压力容器校验。
*   **劳动法**: 劳务派遣<10% 红线/同工同酬/工伤责任/加班时长合规。
*   **环保法**: 垃圾分类强制标准/清运五联单/有害垃圾专处/排污许可。

### 3. 🔗 完整质询网络
17 位专家间建立 **20+ 条强制质询链路**，确保跨域业务零盲区。如 SVC 可质询 FIN (账单争议)、ASSET (房产信息)；ENG 可质询 ASSET (设备位置) 等。

### 4. 📈 铁腕绩效机制
*   **零废话**: 汇报必须基于文件指纹和 Git 记录。
*   **每日考核**: 用户拥有绝对打分权，低于 60 分红牌罚下。
*   **独立归档**: 绩效日报写入 `reports/` 目录，保持主文档清晰。

### 5. 📉 知识卡片化 & Token 精简
*   每位专家浓缩为 50-80 行高密度知识卡片 (表格/公式/符号)。
*   核心技能文件总大小从 ~90KB 压缩至 ~72KB (减少 20%)。
*   信息密度提升 300%+，决策效率大幅提升。

## 💡 调用指南 (Usage)

通过自然语言指令即可唤起专家团队工作：

*   **单点深度评审**:
    `让"合规风控专家"评估装修流程中的舞弊风险。`
*   **圆桌交叉会审**:
    `召集"财务"、"客服"、"HR"专家，圆桌会审"停车费涨价及排班调整"方案。`
*   **全员会诊**:
    `PMO 请出列，针对"新视窗 V4"的公摊收费模块，全员进行一次风险扫描并生成需求文档。`
*   **汇报绩效**:
    `PMO，汇报今日绩效日报。` (将结果写入 `reports/` 新文件并推送)
