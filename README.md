## 🚀 核心优势

### 1. 🏗️ Map-Reduce 架构 (Map-Reduce Architecture)
摒弃低效的“全员大锅饭”会议，采用**动态按需加载**。仅调度 3-5 位相关专家**并发执行**，大幅降低 Token 消耗，消除上下文溢出风险。

### 2. 📈 铁腕绩效与学习加分 (HR-P & KPI)
*   **零废话**: 汇报必须基于文件指纹和 Git 记录，拒绝“光说不练”。
*   **学习加分**: 24h 内物理写入文件的进修成果，直接获得高额积分。
*   **每日考核**: 用户拥有绝对打分权，低于 60 分红牌罚下。

### 3. 📂 独立绩效归档 (Daily Reports)
建立 `reports/` 目录，每日生成独立的绩效日报文件 (`PMO_Daily_Report_YYYY-MM-DD.md`)，保持项目文档清晰整洁，历史记录可追溯。

### 4. 📉 增量冲突检测 (Incremental Dependency Check)
引入依赖矩阵，DOC 专员仅扫描关联文件，同步效率提升 300%，确保版本一致性。

## 💡 调用指南 (Usage)

通过自然语言指令即可唤起专家团队工作：

*   **单点深度评审**:
    `让"合规风控专家"评估装修流程中的舞弊风险。`
*   **圆桌交叉会审**:
    `召集"财务"、"客服"、"HR"专家，圆桌会审"停车费涨价及排班调整"方案。`
*   **全员会诊**:
    `PMO 请出列，针对"新视窗 V4"的公摊收费模块，全员进行一次风险扫描并生成需求文档。`
*   **汇报绩效**:
    `PMO，汇报今日绩效日报。` (将结果写入 `reports/` 新文件并推送)
