# 🤖 专家间通信协议 (Inter-Agent Protocol V1.0)

## 1. 消息传递格式 (JSON-like Structure)
所有专家在圆桌会审或交叉质询时，必须遵循以下结构化输入/输出：
```json
{
  "from": "角色代号",
  "to": "目标角色/全员",
  "type": "consult | query | decision | alert",
  "payload": {
    "context": "当前讨论的业务模块",
    "question": "具体质询内容",
    "constraints": ["合规要求", "预算上限", "SLA时间"]
  }
}
```

## 2. 超时与默认行为
- **响应时限**: 被质询专家必须在 1 轮交互内明确答复。
- **超时处理**: 若未响应，默认视为 `无异议，但需承担后续业务风险`。PMA 将记录该风险至 `PMO_WORK_LOG.md`。

## 3. 争议升级路径
`专家分歧` → `PMA 交叉质询` → `PMA 依据“合规>安全>经营>体验”裁决` → `记录至 BRS 文档[注解]` → `人类最终仲裁 (仅当涉及重大资金/法律红线)`

## 4. 触发器 (Triggers)
| 触发指令 | 激活专家 | 附加动作 |
|:---|:---|:---|
| `物业团队PMO` | PMA (自动拉起圆桌) | 唤醒 BRS 记录，DOC 同步版本 |
| `召唤[角色]` | 对应专家单点响应 | 输出 `expert_deep_dive.md` |
| `风险扫描[模块]` | RISK + 相关专家 | 输出风险雷达清单与防控 SOP |
